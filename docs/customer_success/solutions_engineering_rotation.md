# Solutions Engineering Coverage (Draft)

## Objectives
- Provide predictable support for design partners during pilots.
- Ensure library-first mindset: guidance focuses on Python APIs, with infrastructure help offered as optional add-ons.

## Rotation Structure
- **Cadence:** Bi-weekly rotation among core contributors (minimum two engineers on call each cycle).
- **Availability:** 8 hours per week reserved for customer sessions, code reviews, or troubleshooting.
- **Channels:** Dedicated Slack thread (for partners) and GitHub issue triage for escalations.

## Responsibilities per Rotation
1. **Kickoff sync:** review active pilots, outstanding actions, and planned releases.
2. **Office hours:** host one 60-minute window for drop-in questions.
3. **Ticket triage:** respond to Sev-2/Sev-3 issues within SLA (see `support_sla.md`).
4. **Knowledge capture:** document learnings as FAQs or playbook updates.

## Handover Checklist
- Summaries of open tickets and next steps.
- Upcoming pilot milestones or deliverables.
- Any blockers requiring product/leadership input.

## Tracking
- Maintain rotation calendar in shared team workspace.
- Log time spent and key outcomes to feed roadmap prioritisation.
